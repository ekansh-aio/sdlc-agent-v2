# 03 — Module Reference

Every file in the project, what it does, key inputs/outputs, and important line references.

## Backend

### Entry Points (Azure Functions)

---

#### `backend/DurableFunctionsHttpStart/__init__.py`

HTTP trigger. The single entry point for all calls from the frontend.

- **Trigger:** POST to `/api/DurableFunctionsOrchestrator` (misleading URL — see `function.json`)
- **What it does:**
  1. Parses JSON request body
  2. Starts a new Durable orchestration instance
  3. Returns the Durable status polling URLs (`statusQueryGetUri`, `sendEventPostUri`, etc.)
- **Returns:** 202 with status URIs on success; 400 on JSON parse error; 500 on orchestration start failure
- **Key detail:** The `instance_id` is auto-generated by Durable Functions — no idempotency key is set, so duplicate clicks from the frontend start duplicate orchestrations.

---

#### `backend/DurableFunctionsOrchestrator/__init__.py`

Orchestrator function. Routes the payload to the right activity.

- **Routing logic:**

```python
if helper_name == "RAS" and activity != "PushtoJira":    → yield ExecuteRAS
if helper_name == "TCG" and activity != "PushtoJira":    → yield ExecuteTCG
if activity == "PushtoJira":                             → yield PushtoJira
if helper_name == "DocIngestionFunction":                → yield DocIngestionFunction
```

- **Input:** Full JSON blob from frontend (passed through unchanged to activity)
- **Output:** Whatever the activity returns

---

#### `backend/ExecuteRAS/__init__.py`

Activity function. Runs the RAS 4-agent chain.

- **Input keys used:** `helper_name`, `requirement`, `input_type`
- **Process:**
  1. Builds `RASAgentBuilder` team
  2. Calls `await team.run_stream(task=requirement)`
  3. Collects all messages; finds last message from `FinalResponseGeneratorAgent`
  4. Strips trailing "TERMINATE"
  5. Builds token usage summary per agent
- **Output:**
  ```json
  {
    "response": "<formatted story text>",
    "chat_history": [{"source": "AgentName", "content": "..."}],
    "agent_token_usage": {"AgentName": {"prompt_tokens": N, "completion_tokens": N}}
  }
  ```
- **Termination:** TextMentionTermination("TERMINATE") OR MaxMessageTermination(26)

---

#### `backend/ExecuteTCG/__init__.py`

Activity function. Runs the TCG 4-agent chain and parses the JSON result.

- **Input keys used:** `issue_type`, `input_type`, `request_data`, `process_type`, `helper_name`
- **Key class:** `TestCaseProcessor`
  - `process_test_issues()` — runs agent team, extracts JSON from last `FinalResponseGeneratorAgent` message
  - Parses `finalData` array from JSON response
  - Converts fields to display-ready format
- **Output:** Same shape as ExecuteRAS but `response` is the JSON string of test cases
- **Termination:** TextMentionTermination("TERMINATE") OR `status == "TERMINATE"` in JSON OR MaxMessageTermination(16)

---

#### `backend/PushtoJira/__init__.py`

Activity function. Pushes AI output to Jira.

- **RAS path:** Calls `JiraClient.update_jira_issue(jira_id, title, description, ac)`
  - Requires: `title`, `description`, `acceptance_criteria`, `jira_id`, `base_url`, `jira_username`, `jira_password`
- **TCG path:** Calls `create_test_issues()` → `JiraTestCaseProcessor`
  - Requires: `request_data` (test case JSON string), `issue_type`, `proj_key`, `jira_id`
- **Returns:** `(status_code, response_message)` tuple
- **Known issue:** TCG return value is a tuple; frontend unpacking at line ~653 of `app.py` may not match this shape.

---

#### `backend/DocIngestionFunction/__init__.py`

One-off / manually triggered activity. Seeds Azure AI Search indexes.

- **Input:** `{ blob_name, container_name, index_name }`
- **Process:**
  1. Downloads CSV from Azure Blob (`STORAGE_ACCOUNT_NAME` + `STORAGE_ACCOUNT_KEY`)
  2. Cleans text rows
  3. Generates embeddings via Azure OpenAI (`text-embedding-ada-002`)
  4. Creates index if missing (with HNSW vector config)
  5. Upserts documents
- **Not wired into main workflow** — must be triggered separately (manually or as a one-off HTTP call)

### Agents

---

#### `backend/agents/ras/agent_builder.py`

Builds the RAS `SelectorGroupChat` team.

- **Class:** `RASAgentBuilder`
- **Method:** `build_team()` — returns configured `SelectorGroupChat`
- **Agents created:** `RequestHandlerAgent`, `AnalyserAgent`, `ReviewerAgent`, `FinalResponseGeneratorAgent`
- **Tools registered:** `semantic_search_ras` on `RequestHandlerAgent`
- **Prompt source:** `PromptManager.get_prompt(ai_helper_name="RAS", agent_name=<name>)` with fallback to `ras_prompts.py` constants

---

#### `backend/agents/tcg/agent_builder.py`

Builds the TCG `SelectorGroupChat` team.

- **Class:** `TCGAgentBuilder`
- **Constructor:** takes `test_type` (`text_manual`, `text_automatic`, `Manual`, `Automatic`)
- **Method:** `build_team()` — returns configured `SelectorGroupChat`
- **Tools registered on RequestHandlerAgent:** conditionally assigns `semantic_search_tcg_manual` or `semantic_search_tcg_cucumber` based on test_type
- **Prompt source:** Same DB → fallback pattern; AnalyserAgent prompt varies by test_type

### Common / Shared Modules

---

#### `backend/common/llm/llm_config.py`

Configures the Azure OpenAI client for AutoGen.

- **Class:** `LLMConfig`
- **Reads from env:** `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_ENDPOINT`, `AZURE_MODEL_DEPLOYMENT_NAME`, `AZURE_CHAT_MODEL_NAME`, `AZURE_OPENAI_API_VERSION`, `AZURE_OPENAI_TEMPERATURE`
- **Returns:** `AzureOpenAIChatCompletionClient` (AutoGen's wrapper)
- **Status:** Working — API key is now set in `.env`

---

#### `backend/common/prompts/prompt_manager.py`

Fetches and manages agent prompts from PostgreSQL.

- **Class:** `PromptManager`
- **Key method:** `get_prompt(ai_helper_name, agent_name)` — returns highest-version `prompt_content` or raises `RuntimeError` if not found
- **Also provides:** `get_prompt_parameter()`, `insert_prompt()`, `delete_prompt()`, `create_table()`
- **Query:** `SELECT prompt_content FROM agent_prompts WHERE ai_helper_name=%s AND agent_name=%s ORDER BY version DESC LIMIT 1`
- **Critical:** If the table is empty or the row doesn't exist, a `RuntimeError` is raised at import time in `tcg_prompts.py` (see bug #1 in component status)

---

#### `backend/common/prompts/ras_prompts.py`

Hardcoded fallback prompts for RAS agents.

Contains string constants:
- `REQUEST_HANDLER_PROMPT` — gateway + semantic search tool usage instructions
- `ANALYSER_PROMPT` — INVEST-based story writing rules
- `REVIEWER_PROMPT` — validation checklist + FAILED/SUCCESS routing
- `FINAL_RESPONSE_GENERATOR_PROMPT` — format-only, no content modification
- `SELECTOR_PROMPT` / `TEAM_PROMPT` — routing logic for `SelectorGroupChat`

These are used when the `agent_prompts` DB table doesn't have a matching row.

---

#### `backend/common/prompts/tcg_prompts.py`

Hardcoded fallback prompts for TCG agents.

- Similar structure to `ras_prompts.py`
- `ANALYSER_MANUAL_PROMPT` — includes 3 full worked examples of manual test cases (AIHQE-300, AIHQE-311, AIHQE-313)
- `ANALYSER_CUCUMBER_PROMPT` — includes 3 full worked cucumber scenarios
- **Bug:** Line 17 calls `PromptManager.get_prompt("TCG", "team_prompt")` at module load time. If the DB row is missing, the entire module fails to import, crashing ExecuteTCG.

---

#### `backend/common/ai_search/azure_ai_search_client.py`

Semantic search client for Azure AI Search.

- **Class:** `AzureAISearchClient`
- **Constructor:** reads `AZURE_SEARCH_ENDPOINT`, `AZURE_SEARCH_ADMIN_KEY`
- **Methods:**
  - `get_embeddings(query)` → calls `text-embedding-ada-002`, returns 1536-dim list
  - `semantic_search_ras(query)` → searches `ras-helper-index-v1`, returns `[{description, summary, acceptance_criteria}]`
  - `semantic_search_tcg_manual(query)` → searches `tcg-manual-index-v1`, returns `[{manual_test_steps}]`
  - `semantic_search_tcg_cucumber(query)` → searches `tcg-cucumber-index-v1`, returns `[{cucumber_scenario}]`
- **Search config:** `VectorizedQuery`, top 3, score threshold 0.85
- **Status:** Working — endpoint and admin key are now set in `.env`

---

#### `backend/common/db/postgres_client.py`

PostgreSQL connection pool for backend.

- **Class:** `PostgresClient`
- **Reads from env:** `POSTGRES_HOST`, `POSTGRES_PORT`, `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`
- **Uses:** `psycopg2` connection pool (simple pool, not async)
- **Used by:** `PromptManager`

#### `backend/common/jira/jira_client.py`

Full Jira REST API v2 client.

- **Class:** `JiraClient`
- **Auth methods:**
  - Basic Auth: `set_credentials(username, password)` → base64 encodes for `Authorization: Basic` header
  - Session cookie: `authenticate_user()` via `/rest/auth/1/session` (legacy fallback)
- **Key methods:**

| Method | Endpoint | Purpose |
|--------|---------|---------|
| `authenticate_user()` | GET `/rest/api/2/myself` | Validate credentials |
| `fetch_issue_details(jira_id)` | GET `/rest/api/2/issue/{id}` | Get summary, description, ACs |
| `update_jira_issue(id, title, desc, ac)` | PUT `/rest/api/2/issue/{id}` | Update story fields |
| `get_accessible_issues(issue_types)` | GET `/rest/api/2/search?jql=...` | List issues (paginated) |
| `create_issue(proj, summary, desc, priority, type)` | POST `/rest/api/2/issue/` | Create Test issue |
| `add_test_step_manual(key, step, data, result)` | PUT `/rest/raven/1.0/api/test/{key}/step` | Add manual test step |
| `add_test_cucumber(key, cucumber_text)` | PUT `/rest/api/2/issue/{key}` | Set cucumber scenario |
| `link_issues(parent, child, link_type)` | POST `/rest/api/2/issueLink` | Link Test → Story |
| `_add_comment(id, text)` | POST `/rest/api/2/issue/{id}/comment` | Add AC as comment |

- **Known bug (line ~384):** `requests.get(..., headers=cookies)` — passes cookie dict as headers. Should be `cookies=cookies`. May cause auth failures on some Jira API calls.

## Frontend

---

#### `frontend/src/app.py`

Main Streamlit application. ~700 lines. Coordinates everything.

**Sections:**
| Lines | Purpose |
|-------|---------|
| 1–34 | Imports, env load, CSS load, session init |
| 35–61 | `placeholder_for_agent()` — HTTP trigger + polling loop |
| 62–120 | Response formatting helpers (markdown cleanup, field bolding) |
| 122–138 | App startup: JiraClient init, auth dialog, sidebar |
| 140–290 | Jira ID input path: fetch issue, display, build payload |
| 291–434 | Free text input path: validate, submit, display response |
| 435–624 | Response display: edit mode, copy, star rating, "Analyse Agents" |
| 625–704 | Push to Jira: confirmation dialog, payload build, result display |
| 705–end | PostgreSQL logging calls |

**Important detail — polling loop (lines 35–61):**
```python
for i in range(70):
    time.sleep(20)
    status = requests.get(status_uri).json()
    if status["runtimeStatus"] == "Completed":
        return status["output"]
```
Maximum wait: 70 × 20 s = 1,400 s (~23 minutes). Azure Durable Functions timeout is 10 minutes — so the frontend can outlive a failed backend.

#### `frontend/src/services/jira_client.py`

Frontend-side Jira wrapper (thin layer over `requests`).

- Mirrors the backend's `JiraClient` but is a separate file
- Used for: authenticate, fetch issue details, get accessible issues
- Auth: Basic Auth via base64 header (same pattern as backend)

---

#### `frontend/src/services/postgres_client.py`

Frontend database client.

- Connects to same PostgreSQL as backend
- Used only for logging (not for prompt reads — that's backend-only)

---

#### `frontend/src/utils/pgdb_operations.py`

Database write operations called by `app.py` after each request.

- `log_user_request(...)` — inserts into `user_request_logs`
- `log_user_feedback(...)` — inserts into `user_feedback`
- `update_edited_response(request_id, edited_text)` — updates `user_edited_response` + sets `is_edited=True`

---

#### `frontend/src/utils/session_manager.py`

Initializes Streamlit session state keys on first load.

Key state variables:
| Key | Type | Purpose |
|-----|------|---------|
| `selected_helper` | str | "RAS" or "TCG" |
| `selected_input_type` | str | "Jira ID" or "Free Text Requirement" |
| `jira_username` | str | Jira login |
| `jira_password` | str | Jira password |
| `jira_selected` | list | Selected Jira IDs |
| `rewritten_content` | dict | jira_id → agent response |
| `pushed_status` | dict | jira_id → bool |
| `free_text_input` | str | Free text entry |
| `jira_auth_popup_actioned` | bool | Dialog completed |
| `jira_user_authenticated` | bool | Credentials valid |
| `response_duration` | float | Seconds for last run |
| `agent_conversation_{id}` | list | Per-issue chat history |

**Critical:** All state is lost on browser refresh — users must re-authenticate.
